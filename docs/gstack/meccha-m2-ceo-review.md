## Key Decisions

### 1. Merge M2 + M3 into one milestone: APPROVED

M2 (player identity + auth) and M3 (territory map) share the same implementation boundary: they both need real API routes, the privacy fix, and the H3→GeoJSON pipeline. Splitting them would mean building the API layer twice. Merge is correct.

### 2. Privacy fix is a prerequisite, not a parallel track: MANDATORY

The current `players` table has:
```sql
create policy "players are readable by everyone"
  on public.players for select
  using (true);
```

This exposes all player columns including `user_id` and `last_active_at` to everyone. Before the dashboard makes `players` reads a normal product feature, this must be corrected. The fix:

- Replace direct `players` SELECT with owner-only + `public_player_profiles` view (display_name, faction, created_at only)
- New migration: `002_m2_m3_player_onboarding_territory.sql` — do NOT modify `001_initial_schema.sql`

This is not optional. It is a prerequisite to declaring M2/3 done.

### 3. API routes are the product surface, not a backend detail: REQUIRED

The following routes must exist and return explicit DTOs (no raw Supabase row passthrough):

| Route | Auth | Turnstile | Purpose |
|-------|------|-----------|---------|
| `GET /api/player/me` | Required | No | Onboarding status + player identity |
| `POST /api/player/onboard` | Required | Yes (middleware) | Faction select + display name |
| `GET /api/dashboard` | Required | No | Player summary + faction standings |
| `GET /api/territory` | Optional | No | H3 cell GeoJSON, public-safe by design |

`/api/territory` must be public-safe by design — no exact coordinates, no `private_location_id`, no `player_id`. The map payload is a GeoJSON FeatureCollection of H3 cell polygons.

### 4. Territory data model: materialized `territory_cells` table

Do NOT derive territory state from `territory_events` at request time in M2/3. Add a `territory_cells` table, seed it for Reading, and expose it through `/api/territory`. M6 recalculation jobs update this table. Frontend contracts stay stable.

Schema:
```sql
territory_cells(h3_cell PK, area_label, controller_faction, state, active_hide_count, contested_hide_count, updated_at)
-- state: 'unclaimed' | 'controlled' | 'contested'
-- RLS: SELECT everyone, INSERT/UPDATE service_role only
```

### 5. MapLibre: client-only component, polygons only

- Use `maplibre-gl` npm package (not CDN) for version control
- `"use client"` directive — never SSR MapLibre
- GeoJSON source with fill + line layers, no markers, no point sources
- H3 `cellToBoundary()` → reverse `[lat,lng]` to `[lng,lat]` → close ring → Polygon GeoJSON
- Cell popup: area label, controller, state, hide counts only
- Style: OpenFreeMap (`NEXT_PUBLIC_MAPLIBRE_STYLE_URL`) default, MapTiler fallback

### 6. Faction choice: locked for the pilot

Faction is set at onboarding and cannot be changed in M2/3. Add a 30-day cooldown if a future change mechanism is needed, but lock it now. Display names must be globally unique and case-insensitive.

### 7. Test foundation: add Vitest + Playwright now

Do not defer tests to M7. Auth flows, RLS boundaries, and map privacy are product behaviour from M2/3 onwards. Add:
- Vitest setup + unit tests for H3→GeoJSON conversion (lat/lng ordering, ring closure)
- Route handler tests for forbidden key leakage in `/api/territory`
- Playwright skeleton: `/dashboard` redirect, public page render, API privacy check
- CI script: `npm run lint && npm run typecheck && npm test`

## Scope Boundaries (DO NOT IMPLEMENT in M2/3)

These belong to M4-M7:
- Media upload (clue photos, proof photos)
- Exact GPS submission (hide deployment flow)
- Moderation workflow and queue
- Territory recalculation jobs (systemd timers)
- Hide detail pages, proof submission, capture claims
- Any feature requiring `private_hide_locations` writes or reads in browser code

## Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Players RLS remains too broad | High (current state) | High | Fix in migration 002 before dashboard ships |

| OAuth E2E blocks CI | Medium | Low | Unit-test mappers, skeleton Playwright, defer live OAuth |
| h3-utils coordinate ordering wrong | High (current state) | High | Fix `cellToBoundary` output ordering, add unit test |
